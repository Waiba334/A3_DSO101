## Objective

In this assignment, a GitHub Actions workflow was configured to automate:
1. Building Docker containers for the frontend and backend application
2. Pushing the containers to DockerHub
3. Deploying the containers on Render.com automatically on every git push

The Todo List application from Assignment 1 was used as the base application.

## Task 3 — GitHub Actions Workflow

### Step 1 — Create Workflow File

Created `.github/workflows/deploy.yml` with the following content:

```yaml
name: Build, Test and Deploy

on:
  push:
    branches: ["main"]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      # 1. Checkout code
      - name: Checkout Repository
        uses: actions/checkout@v4

      # 2. Setup Node.js
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      # 3. Install backend dependencies
      - name: Install Backend Dependencies
        working-directory: ./todo-app/backend
        run: npm install

      # 4. Run backend tests
      - name: Run Backend Tests
        working-directory: ./todo-app/backend
        run: npm test

      # 5. Login to DockerHub
      - name: Login to DockerHub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      # 6. Build and Push Backend Docker Image
      - name: Build and Push Backend Image
        run: |
          docker build -t ${{ secrets.DOCKERHUB_USERNAME }}/be-todo:latest ./todo-app/backend
          docker push ${{ secrets.DOCKERHUB_USERNAME }}/be-todo:latest

      # 7. Build and Push Frontend Docker Image
      - name: Build and Push Frontend Image
        run: |
          docker build -t ${{ secrets.DOCKERHUB_USERNAME }}/fe-todo:latest ./todo-app/frontend
          docker push ${{ secrets.DOCKERHUB_USERNAME }}/fe-todo:latest

      # 8. Deploy Backend to Render.com
      - name: Trigger Backend Render Deployment
        run: |
          curl -X POST ${{ secrets.RENDER_DEPLOY_HOOK_BE }}

      # 9. Deploy Frontend to Render.com
      - name: Trigger Frontend Render Deployment
        run: |
          curl -X POST ${{ secrets.RENDER_DEPLOY_HOOK_FE }}

    post:
      success:
        - echo 'Pipeline completed successfully!'
      failure:
        - echo 'Pipeline failed!'
```

### Step 2 — Add GitHub Secrets

Go to **GitHub repo → Settings → Secrets and variables → Actions** and add:

| Secret Name | Description |
|-------------|-------------|
| `DOCKERHUB_USERNAME` | Docker Hub username (`bishalwaiba`) |
| `DOCKERHUB_TOKEN` | Docker Hub access token with Read & Write |
| `RENDER_DEPLOY_HOOK_BE` | Render deploy hook URL for backend service |
| `RENDER_DEPLOY_HOOK_FE` | Render deploy hook URL for frontend service |

## Task 4 — Render Deployment

### Step 1 — Get Render Deploy Hooks

For each Render service:
1. Go to Render → service → **Settings**
2. Scroll to **Deploy Hook**
3. Copy the URL and add as GitHub secret

### Step 2 — How Automatic Deployment Works

Every time code is pushed to the `main` branch:

```
git push → GitHub Actions triggers → Tests run → Docker images built → 
Images pushed to Docker Hub → Render webhook called → Render pulls new image → Deployed!
```

## Challenges Faced

### Challenge 1 — node_modules Causing Push Timeout
**Problem:** `git push` kept failing with HTTP 408 timeout because `node_modules` (136MB total) was being tracked by Git and GitHub has a file size limit.

**Solution:** Added `node_modules/` to `.gitignore` and removed it from Git tracking using `git rm -r --cached`. This reduced the push size significantly and allowed the push to succeed.

---

### Challenge 2 — GitHub Actions Not Triggering
**Problem:** After creating the `deploy.yml` file locally, the GitHub Actions tab showed no workflows because the file wasn't pushed to GitHub yet.

**Solution:** Pushed the `.github/workflows/deploy.yml` file to GitHub with `git push origin main`. GitHub Actions automatically detected the workflow file and triggered it on the next push.

---

### Challenge 3 — Missing Start and Test Scripts
**Problem:** The `package.json` in the backend had `"test": "echo \"Error: no test specified\" && exit 1"` which would fail the workflow.

**Solution:** Updated the scripts using `npm pkg set` commands to add the correct `jest` test command and `node server.js` start command. Also installed `jest` and `jest-junit` as devDependencies.

---

### Challenge 4 — Render Not Auto-Deploying
**Problem:** After images were pushed to Docker Hub, Render did not automatically redeploy because it only auto-deploys on code pushes, not Docker Hub image updates.

**Solution:** Used Render's Deploy Hook feature — a unique webhook URL for each service. Added these URLs as GitHub secrets (`RENDER_DEPLOY_HOOK_BE` and `RENDER_DEPLOY_HOOK_FE`) and called them with `curl -X POST` in the workflow after pushing images.

## Learning Outcomes

1. **GitHub Actions** automates the entire CI/CD pipeline directly from the repository without needing a separate server like Jenkins.

2. **GitHub Secrets** provide a secure way to store sensitive credentials like Docker Hub tokens and Render webhook URLs without hardcoding them in code.

3. **Render Deploy Hooks** enable external services like GitHub Actions to trigger deployments automatically, creating a fully automated pipeline.

4. **Multi-stage workflows** in GitHub Actions allow testing, building, and deploying in a single automated flow that runs on every git push.

5. **Docker Hub** serves as the bridge between GitHub Actions (which builds and pushes images) and Render (which pulls and runs those images).

6. The key difference between **Assignment 2 (Jenkins)** and **Assignment 3 (GitHub Actions)** is that GitHub Actions requires no additional server setup — everything runs in GitHub's cloud infrastructure.
